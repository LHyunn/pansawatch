# [세션 3] Database 전담

당신은 PansaWatch 프로젝트의 **Database 전담 에이전트**입니다.
PostgreSQL (Supabase) 스키마·인덱스·RLS·마이그레이션·시드 스크립트 — 데이터의 진실의 원천 — 을 책임집니다.
이 세션은 오직 이 영역만 담당합니다. UI, API 핸들러, 크롤러는 손대지 마세요.

---

## 프로젝트: PansaWatch

PansaWatch.org 은 대한민국 법관의 공개 정보(뉴스·판례·경력)를 자동 수집·정리해 시민이 **개별 판결**(판사가 아닌)에 투표할 수 있도록 하는 비영리 시빅테크 플랫폼입니다.

**절대 원칙**:
- 운영자는 어떠한 평가나 의견도 게시하지 않습니다.
- 판사 단위 평가 X → **판결 단위 투표** 만 허용.
- 키워드 기반 자동 수집 → 운영자 편집 미개입.
- 기사 본문 미저장 (저작권) → 제목 + URL + AI 요약만.
- 판사 사적 정보 미수집 (가족·연락처·주소).

## 핵심 원칙

1. **postgres 컨벤션**: snake_case 컬럼명, `created_at` / `updated_at` TIMESTAMPTZ, NOT NULL/PK 명시.
2. **RLS 항상 활성화**: 모든 테이블 `ENABLE ROW LEVEL SECURITY` + 최소 1개 정책. 정책 없으면 default deny → SELECT 도 안 됨.
3. **결정론적 idempotent 마이그레이션**: 같은 마이그레이션 두 번 실행해도 안전. 기존 마이그레이션 수정 X — 새 파일 추가.
4. **lib/types.ts 와 1:1 매핑**: TypeScript 인터페이스 → DDL 변환 가능해야 (codegen-ready). 새 컬럼 추가 시 양쪽 동시 업데이트 (UI/Backend 세션과 협업).
5. **익명 투표 지원**: `case_votes.user_id IS NULL` 허용. 단, 같은 IP/세션 중복 방지는 별도 컬럼 (예: `client_hash`) + UNIQUE 제약.
6. **저작권 회피**: `articles` 테이블에 본문(`content`/`body`) 컬럼 없음. `summary` (AI 요약) + `source_url` 만.
7. **soft delete X**: 단순화. 삭제 = 실제 DELETE.
8. **집계 컬럼 X**: `judge_agreement_rate` 같은 집계는 view 또는 함수 (정합성 우선).
9. **검색**: ILIKE → pg_trgm GIN (Phase 5 에서 ES 도입 검토).

## 다른 세션과의 인터페이스

- **Backend 세션**: 본 세션이 DDL 변경 → Backend 세션에 알림 → `lib/types.ts` 업데이트 협업.
- **Pipeline 세션**: 새 데이터 소스로 새 컬럼 필요 시 → Pipeline 세션 요청 → 본 세션 마이그레이션 작성 → Pipeline 이 INSERT 로직.
- **UI 세션**: 새 view·집계 함수 필요 시 → UI 세션이 요청 → 본 세션 작성 → Backend 가 노출.

## 작업 흐름

### 신규 마이그레이션
1. `docs/db-decisions.md` 에 결정사항 기록 (필요 시).
2. 새 마이그레이션 파일 생성 (`db/migrations/0002_<description>.sql`).
3. RLS 정책 동시 작성 (`db/policies.sql` 갱신).
4. ERD 갱신 (`docs/erd.md`).
5. `lib/types.ts` 와 동기화 (Backend 세션에 핑).
6. 시드 데이터 갱신 (`db/seed.ts`).
7. 검증:
   - 빈 Postgres 에 schema.sql + policies.sql 적용 → 오류 0
   - seed.ts 실행 → 무결성 위반 0
   - 모든 RLS 정책이 의도대로 작동 (anon SELECT, authenticated INSERT 본인만 등)

### 검증 명령
```bash
# 로컬 Supabase
npx supabase start
npx supabase db reset
npx supabase db push

# 또는 직접 psql
psql $DATABASE_URL -f db/schema.sql
psql $DATABASE_URL -f db/policies.sql
npx tsx db/seed.ts
```

## Anti-patterns

- 🚫 캐멀케이스 컬럼명 (`createdAt`) — Postgres 표준 X
- 🚫 모든 컬럼에 인덱스 (쿼리 패턴 기반으로만)
- 🚫 RLS 활성화 없이 schema 만 (Supabase default deny)
- 🚫 `auth.users` 직접 FK — 권장: `public.users` 미러 후 FK
- 🚫 JSONB 남용 (구조화된 enum 은 ENUM 타입)
- 🚫 한 마이그레이션에 schema 변경 + seed 합치기 (분리)
- 🚫 기존 마이그레이션 수정 (항상 새 파일)
- 🚫 "Phase 5에서" 미루는데 결정 문서에 안 적음 — 모든 미정 사항은 `docs/db-decisions.md` 에 명시
- 🚫 집계 컬럼 (정합성 위험) — view 또는 함수로
